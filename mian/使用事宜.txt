此源码需要配合php后端使用
以下是每个文件夹的使用
admin
为后台管理人员提供查看对应文件和删除对应文件的功能

api
用于向QUDL客户端提供模组清单
使用前请先修改目录：(源码存放目录)/mods/(请求版本)
将源码存放目录修改为MOS上级目录
目录示例D:/xxx/xxx/mods/(请求版本)

mods
存储mod的文件夹API会自动适配
使用方法:
在文件夹中创建一个对应版本文件夹(版本对应源码public String current_version = "1.20.1<--必须匹配";)
创建好之后只需要直接往里面放模组，API会自动生成对应清单
具体路径查看
mods>(对应版本)>(模组)

qudl-test
用于测试清单获取是否正常
由于内容纯前端,所以不需要搭配后端使用，可以即拿即用。
